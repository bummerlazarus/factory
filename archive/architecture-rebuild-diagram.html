🎯 Five Core Pillars

Pillar 1: Rich Data Capture — Sessions, activity, logs all captured for later analysis
Pillar 2: One Source of Truth — Goals, values, KPIs, frameworks in Supabase + Notion
Pillar 3: SOPs as Skills — Versioned, living documents that improve over time
Pillar 4: Self-Improving Loops — Agents propose pattern promotions; knowledge compounds
Pillar 5: Proactive Surfacing — Daily briefing, nudges, observations without prompting

🔹 Input Layer

Four capture pathways funnel into one capture() Edge Function. Accepts text, URLs, files, and voice memos. One inbox, three surfaces.

🔷 Core: Supabase

PostgreSQL for structured data, Storage for blobs, Edge Functions for business logic, Realtime for live updates, vector layer for semantic search (Pinecone or pgvector Q2).

🔸 Agent Runtime

Claude with frozen-snapshot memory pattern. Each agent gets snapshot at session start; mid-session writes are durable but don't mutate snapshot. Prefix caching stays stable.

🟡 Output Surfaces

Dashboard (Vercel), Notion (ops source-of-truth), scheduled tasks (daily briefing). Realtime bridges all surfaces for instant updates.

💾 Skills Library

SOPs become versioned Skills (voice/tone, content strategy, lead magnets). Session retro → promotion workflow lets agents suggest updates. Skills depend on specific MCPs and docs.

🔌 Native MCPs

Supabase, Pinecone, Notion, Firecrawl. No custom runtime. Claude is the only agent. Architecture stays simple and upgradeable.
