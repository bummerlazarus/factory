## Decisions made

- **Anon SELECT on `work_log`.** Widened the policy. Acceptable because single-user project; revisit if multi-tenant. Alternative (authenticated JWT) would require a real login flow on the dashboard which isn't built.
- **Client-side filter on `kind ∈ {note, research}`.** Realtime `postgres_changes` filter doesn't support arbitrary column filters on value sets (only equality / or via RLS). Cheap to filter client-side — payload is small.
- **10s fade on NEW badge.** Short enough to not accumulate noise; long enough for Edmund to notice a new arrival.
- **Singleton browser client.** `lib/supabase-browser.ts` is imported by any client component that needs Realtime. Prevents opening multiple WebSocket connections per page.
- **No reconnection / error-state UI.** supabase-js handles reconnects automatically. If it fails silently, the list just stops updating (initial SSR rows remain). Acceptable for v1; add connection-state indicator later if it becomes a pain point.
