`kind=url` captures now prefer Firecrawl `/v1/scrape` (when `FIRECRAWL_API_KEY` is configured) for clean page-body markdown, which is embedded into `memory.content`. Bare-fetch HTML-title extraction remains as fallback — so URL captures still work with only `OPENAI_API_KEY`, or with neither key (memory skipped).

New `UrlEnrichment.enrichment_source` field (`"firecrawl" | "bare_fetch"`) reports which path ran, echoed into `work_log.artifacts[0]` and the response envelope.

Markdown body stays in `memory.content` (embeddable, queryable via `match_memory`) — **not** in `work_log.artifacts` (row size hygiene) and **not** in the HTTP response (only `markdown_len` is returned).

## Decisions made

- **Firecrawl first, bare-fetch fallback.** Never throws on Firecrawl error — logs a warn, falls back. URL captures never fail because of enrichment.
- **15s timeout on Firecrawl.** Anthropic-news-level pages complete in ~2s empirically; 15s covers slow targets (e.g. content-heavy blogs) without pinning Edge Function memory unreasonably.
- **Strip markdown from the response.** Keeps captures snappy in dashboard/chat — markdown is large and the caller already has it via `memory.content` if they need it. `markdown_len` is enough signal.
- **Keep work_log artifact light.** `markdown_len` tracks presence; full body stays in memory. Work_log is log, memory is corpus.
- **Dual-read instrumentation not wired in capture.** Capture is a write path. Dual-read is a read-path concern (W1.2 callers). The `dualread.ts` helper is there when needed.

## Follow-ups flagged

- **`FIRECRAWL_API_KEY` needs to land in Supabase function secrets.** One-time config by Edmund. Without it, URL captures still succeed but enrichment_source stays `bare_fetch`.
- **Firecrawl v1 vs v2.** Used `/v1/scrape` (matches the GravityClaw pattern in `reference-repos/gravityclaw/.../web-clip/route.ts`). Firecrawl v2 exists and may have richer metadata; swap later if desired.
- **Error surface.** If Firecrawl returns 200 but `success: false`, we fall back to bare-fetch. If Firecrawl returns a non-200 (e.g. 402 insufficient credits), we also fall back. Consider logging these distinctly once real usage arrives so we notice quota exhaustion.
- **onlyMainContent.** Could be toggled to capture full-page HTML (`formats: ["html"]`) later if the main-content extractor under-selects for Edmund's typical URLs.
