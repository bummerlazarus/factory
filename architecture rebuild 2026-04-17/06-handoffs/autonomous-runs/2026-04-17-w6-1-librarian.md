## Summary

Librarian ships as a single Postgres function `public.librarian_cluster_observations(window_days int)` scheduled via pg_cron daily at 06:00 UTC. Clusters observations by embedding cosine similarity (≥ 0.85) via their paired `memory` rows; writes one `reference_docs(kind='cluster')` row per non-singleton connected component. No Edge Function, no external trigger.

## What was applied

Migration `20260417214428 / librarian_cluster` (file: `dashboard/supabase/migrations/20260417060000_librarian_cluster.sql`):

1. `CREATE EXTENSION IF NOT EXISTS pg_cron` (new).
2. `ALTER TABLE reference_docs` — extended `kind` CHECK to include `'cluster'`.
3. `CREATE OR REPLACE FUNCTION public.librarian_cluster_observations(int)` — SECURITY DEFINER, search_path=public. Returns jsonb.
4. `cron.schedule('librarian-daily', '0 6 * * *', ...)`.

Idempotency: migration is safe to re-run — `CREATE EXTENSION IF NOT EXISTS`, `DROP CONSTRAINT IF EXISTS` before re-add, `CREATE OR REPLACE FUNCTION`, and `unschedule` guarded by exception handler before re-schedule. The function itself upserts on slug.

## Issues hit + fixed mid-run

**`min(uuid)` isn't a defined aggregate.** First function version hit `function min(uuid) does not exist` during the connected-components CTE. Fixed by casting to text for aggregation and back: `(min(node::text))::uuid AS component_key`. Migration file was updated to match. Applied in-database via `CREATE OR REPLACE FUNCTION`.

## Trigger shape

- **What runs it:** pg_cron job `librarian-daily` inside the Supabase Postgres instance.
- **When:** `0 6 * * *` — 06:00 UTC daily.
- **Default window:** 7 days.
- **Manual run:** `SELECT public.librarian_cluster_observations(<days>);` (any caller with `execute` on the function; SECURITY DEFINER so it runs as the owner).
- **No external secret / no Edge Function.**

## Follow-ups

1. **No upstream writes observations yet.** Table is empty; `memory.source='observation'` is 0 rows. The Librarian runs on nothing. For it to bite, capture/skills/agents need to start emitting observations with matching memory embeddings. Candidate: extend `capture()` to emit observations when it detects a pattern/todo/risk, or schedule a daily "session → observation" summarizer.
2. **Crude cluster titles.** Currently "first 80 chars of exemplar body." Fine as a placeholder — W6.2 can UPSERT the same slug with a better LLM-written title.
3. **Threshold 0.85 is an untuned guess.** Revisit once there's real volume.
4. **No decay.** Old cluster reference_docs aren't expired. Add a maintenance task later if it gets noisy.
