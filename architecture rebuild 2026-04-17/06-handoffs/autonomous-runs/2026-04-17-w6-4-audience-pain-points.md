## Follow-ups

1. **Comment embedding pipeline** — highest priority v2 work. Without embeddings, semantically-similar comments that don't share keywords go uncaught. Proposal: Edge Function (or nightly SQL) that embeds new YouTube comments into `memory` with `source='youtube_comment'`, then a v2 of this Skill joins through `memory` and uses the same pairwise-cosine pattern as W6.1 Librarian. Lexical v1 is good enough for the 100-comment regime; we'll feel the gap when comment volume scales.
2. **`/inbox/research` UI** — not built yet. Separate follow-up epic. Reads `reference_docs WHERE kind IN ('cluster','pain-point-cluster')`.
3. **DM ingest** — unblocks when an alternate Instagram/DM source comes online (W5.4 alternative).
4. **Community threads** — unblocks when Q12 (Circle as canonical community) resolves.
5. **Noise filters** — at scale (>5k comments), add content-length gate and emoji/spam drop before regex matching.
