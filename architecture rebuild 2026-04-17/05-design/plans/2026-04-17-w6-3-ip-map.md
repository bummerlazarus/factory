# W6.3 — IP Map doc generator (plan)

**Epic:** W6.3 — living "IP map" reference_doc regenerated weekly.
**Date:** 2026-04-17
**Status:** draft → ship-ready
**Pattern:** sibling to W6.2a (research-director themes), W5.8 (double-down), W6.4 (pain-points) — Skill-as-SQL-function + pg_cron.

## 1. Why

Edmund's Working Genius is Wonder / Invention. He spawns frameworks faster than he authors them. Today the only way to know what IP he has vs. gestures at is to eyeball `reference_docs` + skim his corpus by memory.

The IP map is a single canonical doc — `reference_docs` row `slug='ip-map'` — that a weekly Postgres job rewrites. Four sections:

1. **Authored** — frameworks with a concrete canonical doc (e.g., IOC v2).
2. **Hinted-at** — themes (`kind='theme'`) that span his corpus but have no matching `framework` row.
3. **Market-asked-for** — `pain-point-cluster` rows from W6.4.
4. **Gaps** — frameworks lacking any concrete artifact + recurring themes with no framework author.

## 2. Guardrails (from the brief)

- No LLM calls. Pure SQL per sibling pattern.
- Additive migrations only. Do NOT drop any existing kinds.
- Do NOT touch `memory` table.
- Do NOT create new dashboard pages.
- Use `reference_docs_kinds` via `INSERT ... ON CONFLICT DO NOTHING` (W9.1c convention).

## 3. Data shape

One row in `reference_docs`:

```
slug      = 'ip-map'
kind      = 'ip-map'
title     = 'IP Map — living snapshot'
body      = rendered markdown (Authored / Hinted-at / Market-asked-for / Gaps)
metadata  = {
  regenerated_at: <timestamptz>,
  section_counts: { authored: N, hinted_at: N, market_asked_for: N, gaps: N },
  generator: 'ip_map_regenerate',
  version_bump: <int>
}
```

Idempotent via `INSERT ... ON CONFLICT (slug) DO UPDATE`. The existing `BEFORE UPDATE` trigger keeps `updated_at` fresh.

## 5. Section builders (SQL)

Each section is built as a small CTE inside the function. All four feed into one `string_agg` that renders the final markdown body.

### 5.1 Authored

Frameworks with a concrete canonical doc (non-trivial body length heuristic — a row tagged `kind='framework'` with `body` ≥ 400 chars OR with metadata indicating it was imported from a source file).

```sql
SELECT slug, title, created_at,
       LENGTH(body) AS body_len,
       metadata->>'source_path' AS source_path
FROM public.reference_docs
WHERE kind = 'framework'
  AND (
      LENGTH(COALESCE(body, '')) >= 400
      OR metadata ? 'source_path'
      OR metadata ? 'imported_at'
  )
ORDER BY created_at DESC
LIMIT 10;
```

Rendered bullet:

```
- {title} — slug `{slug}`, {body_len} chars{source_path ? ", from " + source_path : ""}
```

### 5.2 Hinted-at

Themes whose title / slug does NOT match any existing framework slug. Uses token overlap between theme title and framework slugs — cheap heuristic: if the theme's top two words (lowercased, alphanumeric only, length ≥ 4) appear in any framework slug or title, consider it "covered". Otherwise it's hinted-at.

```sql
WITH framework_terms AS (
    SELECT DISTINCT lower(unnest(regexp_matches(
        concat_ws(' ', slug, title), '[a-z0-9]{4,}', 'g'
    ))) AS term
    FROM public.reference_docs
    WHERE kind = 'framework'
),
theme_terms AS (
    SELECT t.slug, t.title, t.created_at, t.metadata,
           array_agg(DISTINCT lower(term) ORDER BY lower(term)) AS terms
    FROM public.reference_docs t,
         LATERAL regexp_matches(
             concat_ws(' ', t.slug, t.title),
             '[a-z0-9]{4,}', 'g'
         ) AS term
    WHERE t.kind = 'theme'
    GROUP BY t.slug, t.title, t.created_at, t.metadata
)
SELECT tt.slug, tt.title, tt.created_at,
       tt.metadata->'member_sources' AS member_sources,
       (tt.metadata->>'cluster_size')::int AS cluster_size
FROM theme_terms tt
WHERE NOT EXISTS (
    SELECT 1 FROM framework_terms ft
    WHERE ft.term = ANY(tt.terms)
)
ORDER BY (tt.metadata->>'cluster_size')::int DESC NULLS LAST,
         tt.created_at DESC
LIMIT 10;
```

Rendered bullet:

```
- {title} — cluster of {cluster_size}, sources: {member_sources}
```

### 5.3 Market-asked-for

`pain-point-cluster` rows from W6.4 in the last 90 days (rolling window; today 0 rows).

```sql
SELECT slug, title, created_at,
       (metadata->>'cluster_size')::int AS cluster_size,
       metadata->>'top_pattern' AS top_pattern
FROM public.reference_docs
WHERE kind = 'pain-point-cluster'
  AND created_at >= now() - interval '90 days'
ORDER BY (metadata->>'cluster_size')::int DESC NULLS LAST,
         created_at DESC
LIMIT 10;
```

### 5.4 Gaps

Gap heuristic — **chosen & justified**:

> **A gap = either (a) a `framework` row with a "thin" body (< 400 chars) AND no linked concrete artifact, OR (b) a `theme` slug that appears across ≥ 2 distinct dated theme rows within the last 30 days (the theme keeps surfacing but no one has authored a framework).**

**Why this heuristic?**

- We have no FK from `framework` → `workspace_items` yet (workspace_items has 0 rows today), so "no concrete artifact" can only be measured via body length + metadata hints. A thin body is a strong signal that a framework slug was created speculatively.
- A theme cluster that persists week-over-week is the textbook "hinted-at long enough to deserve authorship" pattern. Same slug ≥ 2 times in 30 days captures that without noise. (Slugs are exemplar-keyed, so a real repeat will almost never share an md5 hash — instead the recurrence shows as *similar* slug prefixes. Simpler and closer to spec: count distinct `theme-YYYY-MM-DD-*` dates where the first word of the title matches.)

Refined SQL:

```sql
-- 5.4a Thin frameworks
WITH thin_frameworks AS (
    SELECT slug, title, LENGTH(COALESCE(body,'')) AS body_len
    FROM public.reference_docs
    WHERE kind = 'framework'
      AND LENGTH(COALESCE(body, '')) < 400
      AND NOT (metadata ? 'source_path')
),
-- 5.4b Recurring themes (first title-word repeats across ≥ 2 distinct weeks in 30 d)
theme_recurrence AS (
    SELECT lower(split_part(regexp_replace(title, '^Theme \d{4}-\d{2}-\d{2} — ', ''), ' ', 1)) AS head_word,
           array_agg(DISTINCT date_trunc('week', created_at)) AS weeks,
           array_agg(DISTINCT slug ORDER BY slug) AS slugs,
           MAX(created_at) AS latest
    FROM public.reference_docs
    WHERE kind = 'theme'
      AND created_at >= now() - interval '30 days'
      AND title LIKE 'Theme %'
    GROUP BY 1
    HAVING COUNT(DISTINCT date_trunc('week', created_at)) >= 2
)
```

Rendered bullets:

```
- [thin-framework] {title} — slug `{slug}`, only {body_len} chars, needs authoring
- [recurring-theme] "{head_word}" — appeared across {n} weeks, no framework yet
```

## 6. Function skeleton

```sql
CREATE OR REPLACE FUNCTION public.ip_map_regenerate()
RETURNS jsonb
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public, pg_temp
AS $$
DECLARE
    v_now timestamptz := now();
    v_authored_md text; v_hinted_md text; v_market_md text; v_gaps_md text;
    v_authored_ct int := 0; v_hinted_ct int := 0; v_market_ct int := 0; v_gaps_ct int := 0;
    v_body text;
BEGIN
    -- Authored --------------------------------------------------------------
    SELECT COALESCE(string_agg(bullet, E'\n' ORDER BY created_at DESC), '_(none)_'),
           COUNT(*)
      INTO v_authored_md, v_authored_ct
    FROM (<section 5.1 CTE mapped to bullets>) sub;

    -- Hinted-at, Market-asked-for, Gaps analogous ...

    v_body :=
        E'# IP Map — living snapshot\n\n'
        || '_Regenerated ' || to_char(v_now, 'YYYY-MM-DD HH24:MI UTC') || E'. Auto-generated by `ip_map_regenerate()` (W6.3)._\n\n'
        || E'## 1. Authored\n\nFrameworks Edmund has committed to paper.\n\n' || v_authored_md || E'\n\n'
        || E'## 2. Hinted-at\n\nThemes spanning his corpus but with no matching framework row.\n\n' || v_hinted_md || E'\n\n'
        || E'## 3. Market-asked-for\n\nPain-point clusters from audience comments (W6.4).\n\n' || v_market_md || E'\n\n'
        || E'## 4. Gaps\n\nThin frameworks + recurring themes with no author.\n\n' || v_gaps_md || E'\n';

    INSERT INTO public.reference_docs (slug, title, body, kind, metadata)
    VALUES ('ip-map', 'IP Map — living snapshot', v_body, 'ip-map',
            jsonb_build_object(
                'regenerated_at', v_now,
                'section_counts', jsonb_build_object(
                    'authored', v_authored_ct,
                    'hinted_at', v_hinted_ct,
                    'market_asked_for', v_market_ct,
                    'gaps', v_gaps_ct
                ),
                'generator', 'ip_map_regenerate'
            ))
    ON CONFLICT (slug) DO UPDATE
      SET title = EXCLUDED.title,
          body = EXCLUDED.body,
          kind = EXCLUDED.kind,
          metadata = EXCLUDED.metadata,
          version = public.reference_docs.version + 1,
          updated_at = now();

    RETURN jsonb_build_object(
        'regenerated_at', v_now,
        'authored', v_authored_ct,
        'hinted_at', v_hinted_ct,
        'market_asked_for', v_market_ct,
        'gaps', v_gaps_ct
    );
END;
$$;
```
