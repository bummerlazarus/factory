## Decisions made

- **Best-effort memory write.** If embed or insert fails, the request still returns 200 with `memory_warning`. Rationale: work_log is the durable artifact; losing the embedding is recoverable by backfilling (`SELECT id FROM work_log WHERE …` then embed + insert). Losing the capture itself is not recoverable without re-prompting the user.
- **Idempotency via `source='capture' + source_id=work_log.id`.** Partial unique index on `(source, source_id)` protects against exact retries of the same request. Different requests with identical content still create separate rows — OK for v1; add content-hash if replay becomes a problem.
- **URL memContent = title + url + caller content.** v6 doesn't fetch page bodies. Good-enough signal for retrieval until W2.1c adds Firecrawl and the full page body gets embedded.
- **Namespace split: `knowledge` for text, `content` for url.** Matches existing Pinecone corpus convention; the research-director layer can filter cleanly.
