### Data migration (attempts #5 + #6)

**Attempt #5 (no NUL fix)** — same 22P05 errors, ended at upserted=12,585. Surfaced the real root cause: 2 batches at offsets 6500 + 6600 (200 rows total) contained `\u0000` bytes that Postgres text type rejects. Migration report showed the errors; previous phase-2 log did not.

**Script patched** — added `stripNulls()` + recursive `stripNullsDeep()` applied in `normalize()` to scrub NUL bytes from content and all metadata strings before upsert. `deno check` clean.

**Attempt #6 (with NUL fix)** — exited cleanly. 1 error remained: batch 8500 got a transient Cloudflare 502 Bad Gateway (unrelated to data). Report: `upserted: 12685, Errors: 1`. The 100 rows from that batch were already in place from a prior attempt (partial unique index = idempotent), so the final live count closed the gap.
